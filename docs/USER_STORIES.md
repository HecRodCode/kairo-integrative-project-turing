## Format

Each story follows the standard format:

> **"As a [role], I want to [action] so that [benefit]."**

Acceptance criteria use the **Given / When / Then** pattern.

## Epic 1 — Personalized Learning (Coder)

### US-01: View Prioritized Learning Cards

> **"As a Coder, I want to see 6 prioritized cards based on my `module_id` so
> that I can practice theory and exercises generated by AI."**

**Priority:** High  
**Tables:** `complementary_plans`, `plan_activities`, `moodle_progress`

**Acceptance Criteria:**

```
Given I am logged in as a Coder
  And my moodle_progress.average_score < 70 OR my autonomy ≤ 2

When I open my Dashboard
Then I see exactly 6 active cards from complementary_plans
  And 2 cards have priority_level = 'high' (displayed first)
  And 2 cards have priority_level = 'medium'
  And 2 cards have priority_level = 'low'
  And each card shows its day-by-day activities from plan_activities
```

**SQL Query (backend):**

```sql
SELECT * FROM complementary_plans
WHERE coder_id = :my_id AND is_active = true
ORDER BY priority_level DESC
LIMIT 6;
```

### US-02: Complete Soft Skills Assessment

> **"As a Coder, I want to complete the 20-question quiz to populate my
> `soft_skills_assessment` and receive a tailored learning style."**

**Priority:** High  
**Tables:** `users` (first_login), `soft_skills_assessment`

**Acceptance Criteria:**

```
Given I log in for the first time (users.first_login = true)

When I am redirected to the Onboarding flow
Then I see a 20-question quiz covering:
  - Autonomy (4 questions)
  - Time management (4 questions)
  - Problem solving (4 questions)
  - Communication (4 questions)
  - Teamwork (4 questions)

When I submit my answers
Then a record is created in soft_skills_assessment
  And learning_style is determined (visual / auditory / kinesthetic / mixed)
  And users.first_login is set to false
  And I am redirected to my Dashboard
```

**Logic:**

```javascript
// Determine learning_style from score distribution
if (visualScore >= 70) learning_style = 'visual';
else if (auditoryScore >= 70) learning_style = 'auditory';
else if (kinestheticScore >= 70) learning_style = 'kinesthetic';
else learning_style = 'mixed';
```

### US-03: Mark Activity as Completed

> **"As a Coder, I want to mark daily activities as done and add a reflection so
> that I can track my progress within each learning card."**

**Priority:** Medium  
**Tables:** `activity_progress`, `plan_activities`

**Acceptance Criteria:**

```
Given I am viewing a card's activity detail

When I click "Mark as complete"
Then activity_progress.completed is set to true
  And activity_progress.completed_at is recorded
  And the card's progress bar updates visually

When I write a reflection
Then activity_progress.reflection_text is saved
```

### US-04: Submit Evidence for an Activity

> **"As a Coder, I want to upload a file or link as proof of completing an
> activity so that my TL can review my work."**

**Priority:** Medium  
**Tables:** `evidence_submissions`

**Acceptance Criteria:**

```
Given I have completed an activity

When I upload a file or paste a link
Then a record is created in evidence_submissions
  And the submission is associated with my coder_id and activity_id
  And my TL can see the evidence in the coder detail view
```

## Epic 2 — Performance Monitoring (TL)

### US-05: Access Performance Dashboard by Clan

> **"As a TL, I want to access the `moodle_progress` table through a dashboard
> to identify coders with low average scores."**

**Priority:** High  
**Tables:** `moodle_progress`, `users` (clan_id), `risk_flags`

**Acceptance Criteria:**

```
Given I am logged in as a TL

When I open the Global Dashboard
Then I see all coders in my clan (users.clan_id = 'Turing' / 'Tesla' / 'McCarthy')
  And each coder shows their moodle_progress.average_score
  And coders with score < 70 have a visual risk indicator
  And I can filter by clan using the clan_id dropdown

When I click on a coder
Then I see their full profile including:
  - soft_skills_assessment scores
  - moodle_progress per module
  - active complementary_plans (their 6 cards)
  - pending risk_flags
```

**SQL View used:**

```sql
-- v_coder_risk_analysis (from schema.sql)
SELECT * FROM v_coder_risk_analysis
WHERE calculated_risk_level IN ('medium', 'high')
ORDER BY average_score ASC;
```

### US-06: Send Proactive Feedback by Clan

> **"As a TL, I want to send notifications to specific clanes using the
> `tl_feedback` table to coordinate weekly assignments."**

**Priority:** High  
**Tables:** `tl_feedback`, `users` (clan_id)

**Acceptance Criteria:**

```
Given I am viewing a coder's profile OR the clan dashboard

When I click "Send Feedback"
Then I can write a message and select a feedback_type (weekly / activity / general)
  And the message is stored in tl_feedback with:
    - coder_id: the target coder
    - tl_id: my own user id
    - is_read: false (default)

When the feedback is sent
Then the coder's notification bell shows a red dot 🔴
  And the count of unread messages increments

When the coder reads the message
Then is_read is set to true
  And the red dot disappears ✅
```

**Broadcast to entire clan:**

```javascript
// Send the same message to all coders in a clan
const clanCoders = await supabase
  .from('users')
  .select('id')
  .eq('clan_id', selectedClan)
  .eq('role', 'coder');

for (const coder of clanCoders.data) {
  await supabase.from('tl_feedback').insert({
    coder_id: coder.id,
    tl_id: tlId,
    feedback_text: message,
    feedback_type: 'weekly',
  });
}
```

### US-07: View AI-Generated Risk Report

> **"As a TL, I want to see automated risk alerts for my coders so that I can
> prioritize who needs immediate attention."**

**Priority:** Medium  
**Tables:** `risk_flags`, `ai_reports`

**Acceptance Criteria:**

```
Given the IA agent (risk_detector) runs its analysis

When a coder's autonomy ≤ 2 AND average_score < 70
Then a risk_flags record is created with risk_level = 'high'
  And an ai_reports summary is generated for the TL
  And the TL sees a visual alert badge next to the coder's name

When the TL resolves the issue
Then risk_flags.resolved is set to true
  And risk_flags.resolved_at is recorded
```
